API - USER

Tecnologias usadas

routes.post('/user', userController.createUser);

Router: Esse objeto é usado para definir as rotas e serve para operar as solicitações HTTP para essass rotas.

Post: É usado para enviar dados para o servidor, e também para criar novas rotas.

userController: Função responsável por processar a solicitação e realizar as ações necessárias.

Essa função é responsavel pela criação de usuario no banco de dados. Usado para enviar dados confidenciais.

routes.get('/users', userController.findUsers);

Get: Usado para recuperar uma informação. Verbo padrão para enviar dados quando submetemos um formulario HTTP.

Basicamente o get é usado para enviar as informações menos sensiveis.

routes.delete('/user/:id', userController.deleteUser);

Delete: Usado para deletar componentes.

deleteUser: pega a função do userController, essa funçao especifica é responsavel por deletar.

routes.put('/user/:id', userController.updateUser);

Put: É usado para atualizar ou modificar recursos no servidor.

Ele atualiza e faz modificações em informações já criadas.

routes.post('/user/authenticated', userController.authenticatedUser);

POST: É usado para atualizar ou modificar recursos no servidor.

'/user/authenticated': A rota será acionada quando uma solicitação POST for feita para a URL com o caminho "/user/authenticated".

userController.authenticatedUser: Isso se refere à uma função controlada que será executada quando a rota dor acessada.

A função controladora é resposavel por lidar com a logica da rota que se trata da autenticação do user.
